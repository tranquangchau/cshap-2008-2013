Software Measurement and Analysis
Syllabus

1. Course Title: Software Measurement and Analysis

2. Duration: From 17-Feb-2014 to 30-May-2014

3. Unit of study: 6 (Number of credits: 4)

4. Course Description:

This Software Measurement and Analysis course provides the basic skills needed to define, capture and analyze software process and product measurement in support of management information needs. 

This course covers the following topics:

1. Basics of measurement, data analysis and reporting

2. The objectives of software project measurement and analysis

3. The relationship between metrics and project processes

4. The relationship between metrics and product quality

5. Measuring and managing risk

6. The software project measurement process

7. Defining software project and product measures

8. Capturing and storing software project and product measures

9. Data analysis techniques

10. Status reporting

11. What to do when the collected data says there are significant project problems

12. Collecting and using information for historical purposes

13. ROI associated with software project measurement and analysis


5. Course Learning Objectives

Upon completing year 2 of this program students will be able to enter the work force as a software engineer for small (project team <10 people) projects. 

As a result of completing this course students will be able to:

1. Describe the importance of measurement for software projects

2. Select the appropriate chart type to present data most effectively

3. Define processes and describe basic process characteristics

4. Define key process, product and project metrics and collect and analyze the relevant data:

· Schedule

· Resources

· Productivity

· Budget

· Earned Value

· Quality

· Risk Management

· Change

· Customer Satisfaction

· Team Morale

· Estimation

· Historical Data

5. Use collected data to manage and control project execution according to the project plan

6. Manage project change using data

7. Manage project risk using data

8. Communicate project status to stakeholders using data

9. Capture and use historical data

10. Define their own metrics and collect and analyze the relevant data, build measure and analyze their own processes

11. Articulate and avoid typical software measurement pitfalls
